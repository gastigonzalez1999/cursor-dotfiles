# Phased delivery workflow (milestones and execution)

## When to use

- Breaking big work into phases with clear outcomes (similar to “ship in slices” / GSD-style execution).
- You want a roadmap, checkpoints, and verification instead of an unstructured todo pile.

## Steps

1. Write a **one-line goal** and **definition of done** visible to a stranger.
2. Split work into **phases** ordered by dependencies and risk; put unknowns early as spikes.
3. For each phase, list **tasks** with clear owners or “AI / human” and estimated size.
4. Add **acceptance checks** (tests, demo script, metrics) before marking a phase complete.
5. After each phase, **verify** against the goal; adjust the next phase if reality changed.
6. Use **checkpoints** for human review on irreversible or product decisions.
7. Keep a running **risks/assumptions** list; close or update items as you learn.
8. Prefer finishing vertical slices over horizontal “all infra then all features” when possible.
